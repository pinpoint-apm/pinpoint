# i18n Rules

## Setup
- Library: i18next + react-i18next
- Languages: English (`en.json`) and Korean (`ko.json`)
- Translation files: `packages/ui/src/constants/locales/`
- Fallback language: English

## Usage in Components
```tsx
import { useTranslation } from 'react-i18next';

const { t } = useTranslation();
// Simple: t('COMMON.NO_DATA')
// With interpolation: t('COMMON.MIN_LENGTH', { minLength: 5 })
```

## Adding Translations
1. Add key to both `en.json` and `ko.json`
2. Use dot notation for sections: `"SECTION.KEY_NAME": "value"`
3. Sections: COMMON, SERVER_MAP, TRANSACTION, ERROR_ANALYSIS, INSPECTOR, CONFIG, etc.
4. Always add to BOTH language files simultaneously

## Conventions
- Key names: SCREAMING_SNAKE_CASE with dots for hierarchy
- Never hardcode user-facing strings â€” always use `t()` function
- For dynamic values, use interpolation: `"GREETING": "Hello, {{name}}!"`
- Korean translations should be natural, not literal machine translation
