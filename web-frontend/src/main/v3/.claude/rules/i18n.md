# i18n 규칙

## 설정
- 라이브러리: i18next + react-i18next
- 언어: 영어(`en.json`)와 한국어(`ko.json`)
- 번역 파일 위치: `packages/ui/src/constants/locales/`
- 폴백 언어: 영어

## 컴포넌트에서 사용법
```tsx
import { useTranslation } from 'react-i18next';

const { t } = useTranslation();
// 단순 사용: t('COMMON.NO_DATA')
// 보간 사용: t('COMMON.MIN_LENGTH', { minLength: 5 })
```

## 번역 추가 방법
1. `en.json`과 `ko.json` 모두에 키 추가
2. 섹션에는 점 표기법 사용: `"SECTION.KEY_NAME": "value"`
3. 섹션: COMMON, SERVER_MAP, TRANSACTION, ERROR_ANALYSIS, INSPECTOR, CONFIG 등
4. 항상 두 언어 파일에 동시에 추가

## 규칙
- 키 이름: 계층 구분을 위해 점을 사용하는 SCREAMING_SNAKE_CASE
- 사용자 노출 문자열을 하드코딩하지 않음 — 항상 `t()` 함수 사용
- 동적 값은 보간 사용: `"GREETING": "안녕하세요, {{name}}님!"`
- 한국어 번역은 자연스러운 한국어로, 직역(기계 번역) 금지
